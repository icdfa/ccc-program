# Case Study: The Equifax Data Breach

## Introduction

The Equifax data breach, which occurred in 2017, was one of the largest and most significant data breaches in history. The breach exposed the personal information of approximately 147 million people.

## What Happened?

-   **Vulnerability:** Equifax failed to patch a known vulnerability in the Apache Struts web application framework.
-   **Exploitation:** Attackers exploited the vulnerability to gain access to Equifax's network.
-   **Lateral Movement:** The attackers moved laterally through the network, accessing and exfiltrating sensitive data.
-   **Data Exfiltration:** The attackers exfiltrated a large amount of data, including names, Social Security numbers, birth dates, addresses, and, in some instances, driver's license numbers.

## Lessons Learned

-   **Patch Management:** The breach highlighted the critical importance of timely patch management.
-   **Vulnerability Management:** Organizations must have a robust vulnerability management program to identify and remediate vulnerabilities.
-   **Network Segmentation:** Proper network segmentation could have limited the attackers' ability to move laterally through the network.
-   **Incident Response:** Equifax's response to the breach was widely criticized, highlighting the need for a well-defined and tested incident response plan.
-   **Third-Party Risk Management:** The breach also raised questions about the security of third-party data aggregators like Equifax.
